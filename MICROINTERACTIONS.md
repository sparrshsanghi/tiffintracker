# Microinteractions

## Interaction Principles

Microinteractions should make the product feel attentive, not playful.

They should confirm what just happened, reduce uncertainty, and quietly reward trust.

## Today Screen

### Closed Tiffin Tap

- Tap opens the daily ritual.
- The action should feel slightly anticipatory.
- If the meal is already open for the day, the tap should simply expand details or breathe the card.

### Meal Refresh

- A change in meal state should trigger a soft re-reveal.
- The new meal should feel freshly packed.

### Delivery Status Change

- Status labels should change with a minimal fade or slide.
- The update should never feel alarming.

## Onboarding

### Conversational Progress

- Each answer should feel like the profile is being understood.
- Use subtle confirmations instead of visible forms.
- The user should feel the system is building itself around their routine.

### Memory Capture

- When a detail is confirmed, it should settle quietly into the profile.
- The UI should reinforce that the app remembered something useful.

## AI

### Suggestion Chips

- Suggestions should appear as supportive prompts.
- They should not look like chatbot noise.

### Confirmation State

- Once the request is understood, the confirmation should feel clear and humane.
- The transition into approval should feel like a handoff, not a technical workflow.

### Clarification

- Ask only what is necessary.
- Clarifications should feel like a thoughtful follow-up, not a failed parse.

## Profile

### Membership Feel

- Profile updates should feel like service records becoming part of the member experience.
- Changes should settle into the page with dignity.

### Subscription Details

- Use a quiet reveal for stable information.
- Avoid dense setting toggles where possible.

## History

### Memory Entries

- New entries should arrive like remembered moments.
- The timeline should feel emotionally neutral but personally meaningful.

### Event Expansion

- Expanding an event should feel like unfolding a memory, not opening a log row.

## Delivery and Status

### Route Movement

- A route line or delivery cue should only move when delivery is active.
- Motion should be subtle and short-lived.

### Completion

- Success states should feel settled rather than flashy.
- A small sense of closure is enough.

## Notifications

### Arrival

- Notifications should enter with softness and clarity.
- Important updates can have slightly more contrast, but never noise.

### Dismissal

- Dismissal should be clean and immediate.
- The user should always feel in control.

## Manager Interactions

Design philosophy only:

- Prioritize speed of reading.
- Make status changes unmistakable.
- Let approvals feel decisive.
- Avoid ornament that slows decision-making.
- Preserve brand warmth in copy and color accents, but keep the interaction model operational.

## Best Microinteraction

The best microinteraction in the product is the moment the tiffin opens and the meal appears.

Every other interaction should support that feeling of care, clarity, and routine.
